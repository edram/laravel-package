## Usage

### When to attribute

**Add the footer whenever the agent had a hand in the staged changes** — regardless of how the work was carried out, what fraction the agent wrote versus a tool it drove, or whether a human reviewed or tweaked it afterward. Don't make it a judgment call: if the agent contributed to what's being committed, the footer goes in. Omit it only when none of the diff is the agent's doing, e.g. committing edits a human made entirely on their own.

This footer is the **only** indicator of AI involvement that belongs in the commit. Do not add phrases like `Generated by AI` or `Written with AI` in the subject, body, or anywhere else.

### Exact value format

Match the identity to the agent that contributed the changes. Do not reuse another provider's identity. Use exactly:

```
Co-Authored-By: <Agent Name> <Provider Email>
```

Select both values from the same row:

| Current agent | Agent Name | Provider Email |
|---------------|------------|----------------|
| Claude | Product + model in title case, e.g. `Claude Opus 4.8`; fall back to `Claude` only when the model is unavailable | `noreply@anthropic.com` |
| Codex | Product + model in title case, e.g. `Codex GPT-5`; fall back to `Codex` only when the model is unavailable | `noreply@openai.com` |

Use the real agent and model currently running; do not use a tool name, username, or made-up label. Wrap the email in angle brackets. If the current provider is not listed, ask for an explicit mapping before committing — never invent an address or borrow one from another provider. The token itself follows the general trailer rules in [features-footers](features-footers.md).

### Examples

```
fix: handle null pointer in user serializer

Closes #88
Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>
```

```
feat: add retry handling to the upload client

Co-Authored-By: Codex GPT-5 <noreply@openai.com>
```

## Key Points

- Required whenever the agent had a hand in the staged changes, regardless of how the work was produced or whether a human reviewed it; omitted only when none of the diff is the agent's doing
- Keep agent name and email provider-aligned: Claude uses `noreply@anthropic.com`; Codex uses `noreply@openai.com`
- Use the actual product + model name when available; fall back only to that provider's product name
- Never use a user address, a per-commit address, or another provider's address
- Multiple agents or human co-authors stack as separate `Co-Authored-By` lines, each following the same exact format
- Keep the subject and body free of AI markers — attribution lives only in the `Co-Authored-By` footer
- See [features-footers](features-footers.md) for the general trailer-block placement rules
